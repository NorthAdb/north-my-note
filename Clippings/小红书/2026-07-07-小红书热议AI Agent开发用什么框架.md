### 实用建议

> **问一问**（AI 自动问答）：
> - **企业级生产、复杂工作流**：首选 **LangGraph**，精确控制节点流转、支持循环和条件跳转
> - **快速搭建多 Agent 协作**：选 **CrewAI**，通过定义"角色"快速组建 AI 团队
> - **知识库/RAG 场景**：选 **LlamaIndex**，文档理解、索引和检索增强生成优势明显
> - **低代码/快速原型**：选 **Dify** 或 Coze
> - **Skills 思路**：把常用功能封装成可复用的技能模块，灵活但可观测性和调试难度较大
> - **LangChain/LangGraph 思路**：显式定义流程，可控性强，适合复杂逻辑

### 关键结论

1. **没有银弹** — 框架选择取决于场景、团队能力和落地目标
2. **LangGraph 是生产级首选** — 在需要高稳定性和可观测性的长流程任务中，LangGraph 是主流选择
3. **Skills 和框架不冲突** — Skills 是能力封装，框架是流程编排，可以结合使用
4. **落地 > 技术选型** — 多个评论强调"能落地就 OK"
5. **自研派 vs 框架派** — 自研灵活但维护成本高，框架省力但有限制
6. **Claude Code 不适合云端部署** — 启动慢、进程管理困难，更适合本地/桌面场景
