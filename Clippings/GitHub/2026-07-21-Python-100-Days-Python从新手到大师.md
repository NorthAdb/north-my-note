## 课程结构（100 天 × 7 大阶段）

```
Day01-20  ── Python 语言基础
Day21-30  ── Python 语言应用（文件、Excel、PDF、邮件、图像等）
Day31-35  ── 语言进阶 + Web前端 + Linux
Day36-45  ── 数据库（MySQL / Hive）
Day46-60  ── Django 全栈开发（15天，最重头的部分）
Day61-65  ── 网络数据采集（爬虫）
Day66-80  ── 数据分析（NumPy / pandas / Matplotlib）
Day81-90  ── 机器学习（10+ 经典算法）
Day91-100 ── 团队项目 + Docker + 部署 + 面试
```

### 各阶段详细内容

| 阶段 | 覆盖内容 |
|------|---------|
| **基础 (Day01-20)** | 环境搭建、变量、运算符、分支/循环、数据结构（列表/元组/字符串/集合/字典）、函数、模块、面向对象 |
| **文件与数据 (Day21-30)** | 文件读写、序列化、CSV/Excel/Word/PPT/PDF、图像处理、邮件短信、正则表达式 |
| **进阶技能 (Day31-35)** | Python 高阶语法、Web 前端（HTML/CSS/JS/Vue）、Linux 基础 |
| **数据库 (Day36-45)** | MySQL（DDL/DML/DQL/DCL）、索引、视图、Python 接入 MySQL、Hive |
| **Django (Day46-60)** | 模型/ORM、Ajax、Cookie/Session、RESTful/DRF、缓存、Celery 异步、测试、部署 |
| **爬虫 (Day61-65)** | requests、XPath/CSS 选择器、多线程/异步、Selenium、Scrapy 框架 |
| **数据分析 (Day66-80)** | NumPy（数组/运算/矩阵）、pandas（Series/DataFrame/清洗/透视）、Matplotlib 可视化 |
| **机器学习 (Day81-90)** | KNN、决策树、随机森林、朴素贝叶斯、回归、K-Means、集成学习、神经网络、NLP 入门 |
| **项目实战 (Day91-100)** | 敏捷开发、Docker、MySQL 调优、电商架构、部署上线、面试题 |

## 相比其他 Python 教程的优势

### 优势一：唯一的中文「完整路线图」

| 对比项 | Python-100-Days | 其他常见教程 |
|--------|----------------|-------------|
| **覆盖面** | 语法 → 文件 → 数据库 → Web → 爬虫 → 数据分析 → ML → 部署面试 | 大多只覆盖其中 2-3 个方向 |
| **学习路径** | 按天编号，Day01 → Day100，明确告诉下一步学什么 | 散落的文章/视频，看完不知道接着学什么 |
| **语言** | 全中文，适合英文文档阅读困难的新手 | 英文优质资源多但有语言门槛 |

> 市面上最接近的竞品是 `freeCodeCamp`（英文交互式）和 `CS-Notes`（面试向），但**没有第二个中文项目能像 Python-100-Days 这样从零覆盖到项目部署**。

### 优势二：结构性碾压

| 资源类型 | 代表 | 问题 |
|---------|------|------|
| **视频教程** | B站/慕课网 | 动辄上百小时，快进/拖拽难以定位 |
| **文档教程** | W3School / 菜鸟教程 | 太像字典，缺乏学习顺序 |
| **书籍** | 《笨办法学Python》《流畅的Python》 | 要么太入门、要么太深入，中间断层 |
| **Python-100-Days** | — | **有编号、有进度、有练习**，像一本书但免费实时更新 |

### 优势三：「选赛道」的设计理念

作者在前言中明确给出了职业方向分析，并建议：

> - ✅ Python 后端开发 / 数据分析 / 数据科学 / 自动化测试
> - ❌ **爬虫工程师（作者明确不推荐此赛道）**

这在新手教程中非常罕见——大多数教程只教技术，不教**选方向**。这个项目让学习者一开始就知道每条路通向哪里。

### 优势四：实战密度高

| 项目类型 | 案例数 | 示例 |
|---------|:------:|------|
| 基础练习 | 10+ | 水仙花数、CRAPS赌博游戏、斐波那契 |
| 小项目 | 5+ | 扑克游戏、工资结算系统 |
| 完整 Web 应用 | 3+ | 博客系统、电商系统、团队协作项目 |
| 数据分析实战 | 5+ | 统计图表、同比环比、相关性分析 |
| ML 实战 | 8+ | 房价预测、分类器、聚类分析 |

### 优势五：配套生态

| 渠道 | 内容 |
|------|------|
| **知乎专栏** | 前 20 天基础内容，适合直接在线阅读 |
| **B站视频** | 部分内容有视频讲解 |

## 局限性（需要注意的地方）

| 问题 | 说明 |
|------|------|
| **部分命令过时** | `pip search` 已失效，个别库用法略有变化。复制命令前建议核对 |
| **非交互式教学** | 这是笔记仓库，不是带自动评分/练习平台的课程 |
| **后期内容深度有限** | ML 部分只覆盖基础算法，不够深入；Django 部分需配合官方文档 |
| **Django 版本可能滞后** | 框架默认值和生态实践会变，生产环境需参考当前文档 |
| **答疑需付费** | 免费 QQ 群已满，深度答疑在付费群 |
| **广度 > 深度** | 100 天覆盖极广，但单个主题深度有限，需要后续专项学习 |

## 适合谁、不适合谁

### ✅ 适合
- **Python 零基础的中文学习者** — 最好的中文入门路径之一
- **想转行但不知道 Python 能做什么的人** — 先看前言里的职业分析
- **有基础但想查漏补缺的人** — 把目录当路线图，挑没学过的部分看
- **需要结构化学习的人** — 有编号有进度的强迫感

### ❌ 不太适合
- **已经会 Python 想深入某个方向的人**（如只想学 ML）— 项目广度优先，深度不够
- **需要英文技术文档训练的人** — 中文教程会延迟你适应英文文档
- **希望有练习自动评判的人** — 这不是 LeetCode 式平台
