## 1️⃣ the-art-of-command-line ⭐162k

> 把命令行变成肌肉记忆

**解决的问题**：教材不教命令行，但工业界的日常就泡在终端里。大部分人只会 `cd`、`ls`、`python main.py`。

**内容**：把命令行技巧从入门到精通整理成一份速查手册，涵盖：
- **基础文件操作**：查找、排序、去重、压缩
- **系统调试**：`top`、`df`、`lsof`、`strace` 等排查命令
- **文本处理**：`awk`、`sed`、`grep` 实战
- **网络工具**：`curl`、`ping`、`traceroute` 等

**怎么用**：不是一次背下来，而是每次打开终端前瞄一眼，一个月后这些操作就会变成手指的本能。

**中文版本**：有完整简体中文翻译（README-zh.md）

## 2️⃣ git-flight-rules ⭐42.5k

> 每次 Git 出问题都在搜，不如一次看透

**解决的问题**：Git 出错时第一反应是百度/Google，搜出来的答案五花八门，试了几条命令反而把仓库搞得更乱。

**内容**：它不是教程，是一本**飞行故障手册**。按场景索引：
- 提交错了想撤销？
- 推到错误分支了？
- 合并冲突搞不定？
- 想要修改历史提交信息？

每个场景直接给出**最短的解决步骤**，不用从头学 Git 原理，遇到什么问题翻什么条目。

**怎么用**：用两次就能摸清 Git 真正的运行逻辑，比看十小时视频教程都管用。

**中文版本**：有完整简体中文翻译（README-zh-CN.md）

## 3️⃣ app-ideas ⭐95.5k

> 把简历上的"熟悉XX"变成"我做过XX"

**解决的问题**：简历写"熟悉 Python"没人信，但写"独立开发了一个抓取全校课程信息的爬虫并自动提醒"就有人追问。问题是——不知道做什么项目。

**内容**：按难度分三级给出上百个实战项目方向（只提需求不写代码）：
- **Tier 1（入门）**：二进制转十进制、边框圆角预览器、计算器等
- **Tier 2（中级）**：天气应用、聊天应用等
- **Tier 3（高级）**：带后端的完整应用

每个项目包含清晰目标、用户故事、可选 Bonus 功能、资源链接。

**怎么用**：挑一个感兴趣的从零实现，所有技术决策自己扛。做完放进简历，面试时有东西可聊。

## 4️⃣ free-for-dev ⭐129k

> 免费拿到开发者需要的所有资源

**解决的问题**：很多人毕业了都不知道 GitHub 上有这样一份清单，整理了开发者能免费使用的所有服务。

**内容**：1500+ 贡献者维护的开发者免费资源大全，覆盖：
- ☁️ **云服务**：各家云厂商 always-free 额度
- 🛠️ **CI/CD**：GitHub Actions、CircleCI 等
- 📊 **监控与日志**：免费额度的 APM 工具
- 🗄️ **数据库**：免费层级的数据库服务
- 📧 **邮件服务**：免费邮件发送
- 🔍 **API 与数据**：各种免费 API
- 🎨 **设计与 UI**：设计资源、图标等
- 🤖 **生成式 AI**：免费 AI API 额度

**怎么用**：学生认证能白嫖的东西这里全列出来了，做项目时不用花钱买服务器。

## 5️⃣ system-design-primer ⭐356k

> 补上系统设计那门没人教的课

**解决的问题**：学校里教写代码，不教设计系统。面试官问"短链接系统怎么设计"，死记硬背的模板扛不住追问。

**内容**：系统设计最全学习资源，包含：
- **核心概念图解**：缓存、负载均衡、数据库分片、CDN、消息队列等
- **常见面试题及解法**：设计短链接系统、聊天系统、搜索引擎等
- **Anki 闪卡**：辅助记忆关键概念
- **学习路径**：从入门到进阶的系统性指引

**怎么用**：不是让你背答案，是让你学会怎么思考。这个能力一旦有了，所有系统设计题都变成同一道题。

**中文版本**：有完整简体中文翻译（README-zh-Hans.md）
