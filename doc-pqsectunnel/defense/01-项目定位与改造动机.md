# 01 — 项目定位与改造动机

> 状态：正文 | 权威源：[CONTEXT.md](../../CONTEXT.md) | 预计篇幅：2 min 讲稿（答辩开场）

---

## 1. 一句话定位

**PQ-SecTunnel** 是基于 [WireGuard](https://www.wireguard.com/) 改造的后量子 VPN：把经典 ECDH 密钥交换换成 **KEM 封装/解封装**，并把 **两次握手** 扩展为带 **Key Confirmation** 的 **三次握手**（Noise_IKpsk1kem）。

用户态工具 **`pqst` / `pqst-quick`**，内核模块 **`pqsectunnel.ko`**——协议身份仍是 WireGuard 风格的 TUN + UDP，密码学内核已换血。

---

## 2. 为什么要做（动机）

| 威胁 | WireGuard 现状 | PQ-SecTunnel 方向 |
|------|---------------|------------------|
| **量子计算机** 威胁 Shor 算法破解 ECDH | X25519 经典椭圆曲线 | **ML-KEM** 等 NIST 后量子 KEM |
| 两次握手缺少显式密钥确认 | Noise_IKpsk2，2-RTT | **Key Confirmation** 第三轮 |
| 国标 / 国密需求 | 国际算法套件 | **SM3 + SM4-GCM** 实验套件 |

**答辩聚焦**（CONTEXT 边界）：KEM 讲 **ML-KEM-512/768**；AEAD/Hash 讲 **SM4-GCM/SM3**；主套件 **`mlkem512-mlkem768-sm4gcm-sm3`**。

不在答辩展开：RKEM（已实现未启用）、McEliece（其它实验套件用）。

---

## 3. 相对 WireGuard 的两项核心改造

### 3.1 密钥交换：ECDH → 双 KEM

| | WireGuard | PQ-SecTunnel |
|---|-----------|--------------|
| 数学基础 | X25519 ECDH | KEM Enc/Dec |
| 长期身份 | Curve25519 静态钥 | **Static KEM**（主套件 ML-KEM-512） |
| 临时密钥 | Curve25519 临时钥 | **Ephemeral KEM**（主套件 ML-KEM-768） |
| 握手贡献 | 单次 DH 共享秘密 | ct₁、ct₂、ct₃ 三次封装共同混链 |

这叫 **双 KEM 握手**（CONTEXT **双 KEM 握手**），不是 RKEM。

### 3.2 握手轮次：2 → 3（1.5-RTT）

| | WireGuard IKpsk2 | PQ-SecTunnel IKpsk1kem |
|---|------------------|------------------------|
| 报文 | Initiation + Response | + **Key Confirmation** |
| RTT | 1-RTT 数据可用 | 1.5-RTT；第三轮显式确认 |

Key Confirmation 用新派生密钥加密空载荷，证明对端确实持有相同 **TK**（CONTEXT **Key Confirmation**）。

时序见 [02-协议总览.md](02-协议总览.md) 与 `docs/1.5-RTT交互时序.png`。

## 4. 系统架构（30 秒版）

```
┌──────────── 用户态 ────────────┐     Netlink      ┌────────── 内核态 ──────────┐
│  pqst / pqst-quick            │ ───────────────► │  pqsectunnel.ko            │
│  · 生成/加载 Static 密钥       │   配置 peer/密钥  │  · Noise 握手 (noise.c)    │
│  · 读 .conf 起隧道             │                  │  · KEM / SM3 / SM4-GCM     │
└───────────────────────────────┘                  │  · 分片 / UDP / 数据面      │
                                                   └────────────────────────────┘
```

- **配置面**：密钥、Endpoint、AllowedIPs → Netlink
- **数据面**：UDP 51820 握手 + 加密 IP 包；**密码运算全在内核**

详见 [07-工程架构与实验验证.md](07-工程架构与实验验证.md)。

## 5. 主套件在七种实验中的位置

编译名 = `{static}-{ephemeral}-{aead}-{hash}`：

**`mlkem512-mlkem768-sm4gcm-sm3`**

| 维度 | 选择 | 答辩意义 |
|------|------|---------|
| Static | ML-KEM-512 | 身份封装，ct 768B |
| Ephemeral | ML-KEM-768 | 更高安全级别临时钥，E_i 1184B |
| AEAD | SM4-GCM | 国密对称加密 |
| Hash | SM3 | 国密哈希/KDF |
| 握手大小 | Init 2068B / Resp 1916B | 需**应用层分片** |

七种套件矩阵与构建方式见 [04](04-密码原语与套件.md)、[07](07-工程架构与实验验证.md)。
