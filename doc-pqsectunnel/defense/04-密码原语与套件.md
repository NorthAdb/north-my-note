# 04 — 密码原语与套件

> 状态：正文 | 权威源：[CONTEXT.md](../../CONTEXT.md) + `messages.h` + `build.sh` + wgtest vars  
> 主套件：`mlkem512-mlkem768-sm4gcm-sm3` | 预计篇幅：2 min 讲稿

## 1. 编译时选型原则

PQ-SecTunnel **无运行时算法切换**。一次 `build.sh -e <name>` 通过 C 宏固定：

- Static / Ephemeral KEM 算法
- AEAD / Hash 原语
- 是否启用应用层分片

实验目录命名：`{static}-{ephemeral}-{aead}-{hash}`（CONTEXT **实验目录**）。

## 2. 主套件配置

来源：`wgtest/wgtest-mlkem512-mlkem768-sm4gcm-sm3/vars`

| 变量 | 值 | 编译宏 |
|------|-----|--------|
| STATIC_KEM | mlkem512 | `-DSTATIC_KEM_MLKEM512` |

## 3. ML-KEM 参数（答辩范围）

| 算法 | 公钥 | 密文 | 共享秘密 | CONTEXT |
|------|------|------|---------|---------|
| **ML-KEM-512** | 800 B | 768 B | 32 B | **ML-KEM-512** |
| **ML-KEM-768** | 1184 B | 1088 B | 32 B | **ML-KEM-768** |

常量定义：`kems/mlkem-512/api.h`、`kems/mlkem-768/api.h`（NIST ML-KEM 参数集）。

### 3.1 主套件中的 KEM 角色

| 角色 | 算法 | 握手字段 | KEM 操作 |
|------|------|---------|---------|
| **Static KEM** | ML-KEM-512 | ct₁（Initiation）、ct₃（Response） | Enc(S_r^pub)、Enc(S_i^pub) |
| **Ephemeral KEM** | ML-KEM-768 | E_i^pub（Initiation）、ct₂（Response） | Gen(E_i)、Enc(E_i^pub) |

Static 与 Ephemeral **独立编译选定**（CONTEXT **STATIC_KEM** / **EPHEMERAL_KEM**），在本套件中为 512 静态 + 768 临时。

内核 API 映射（`noise.c`）：

| 用途 | Static 512 | Ephemeral 768 |
|------|-----------|---------------|
| keygen | — | `kem768_keygen`（`pqst_ephemeral_keygen`） |
| encaps | `kem_encapsulate` | `kem768_encapsulate` |
| decaps | `kem_decapsulate` | `kem768_decapsulate` |

`kem/` 构建 symlink 指向 `kems/mlkem-512/`（Static）；Ephemeral 768 通过 `kems/mlkem-768/kem_768.h` 独立引入。

## 4. 国密对称原语

### 4.1 SM3（Hash / KDF）

| 项 | 值 | CONTEXT |
|----|-----|---------|
| 输出长度 | 256 bit（32 B） | **SM3** |
| 编译宏 | `NOISE_HASH_SM3` | |
| 内核实现 | Linux `crypto_shash("sm3")` | `noise.c` `wg_noise_init()` L179–246 |
| 用途 | C/H 链 `mix_hash`；KDF 的 HMAC；`digest_pk` 公钥哈希 | |

`mix_hash()` 在 SM3 路径调用 `sm3_hash_2()`（`noise.c` L837–839）。

### 4.2 SM4-GCM（AEAD）

| 项 | 值 | CONTEXT |
|----|-----|---------|
| 密钥长度 | 128 bit（16 B） | **SM4-GCM** |
| 认证标签 | 16 B | |
| 编译宏 | `NOISE_AEAD_SM4GCM` | |
| 内核实现 | Linux `crypto_aead("gcm(sm4)")` | `noise.c` `message_encrypt/decrypt` L879–960 |
| nonce | 12 字节全零（Noise_IK 约定） | |
| AD | 当前 Hash 值（32 B） | |

`messages.h` L46–48：`NOISE_SYMMETRIC_KEY_LEN=16`，`NOISE_AUTHTAG_LEN=16`。

数据面发送同样走 SM4-GCM（`send.c` L26 起 AEAD 分支；`receive.c` L311 解密分支）。

### 4.3 AEAD/HASH 不影响 KEM 字段长度

握手报文尺寸**仅由 KEM 公钥/密文长度决定**。将 AEAD 从 AES-GCM 换为 SM4-GCM、Hash 从 SHA3 换为 SM3，**不改变** ct₁/ct₂/E_i^pub 等字段字节数；仅对称密钥宽度（128 vs 256 bit）与 Hash 实现路径不同。

## 5. Noise 协议标签

模块加载时 `wg_noise_init()` 预计算：

```
C₀ = SM3(NOISE_HANDSHAKE_NAME)
H₀ = SM3(C₀ ‖ NOISE_IDENTIFIER_NAME)
```

主套件具体字符串：

| 宏 | 值 |
|----|-----|
| NOISE_HANDSHAKE_NAME | `Noise_IKpsk1kem_SM4GCM_SM3_EphemMLKEM768` |
| NOISE_IDENTIFIER_NAME | `PQWireGuard v2 zx2c4 Jason@zx2c4.com` |
| NOISE_MAC1_KEY_LABEL | `mac1----` |

写入方式：`build.sh` 通过 `KCFLAGS -DNOISE_HANDSHAKE_NAME='\"...\"'` 传入编译（`build.sh` L717）。

## 6. 握手报文 struct 与尺寸

定义文件：`wireguard-linux-compat/src/messages.h`。

### 6.1 尺寸宏（主套件）

| 宏 | 值 | 来源 |
|----|-----|------|
| `NOISE_EPHEMERAL_PUBLIC_KEY_LEN` | 1184 | `KEM768_PUBLIC_KEY_SIZE` |
| `NOISE_INITIAL_CIPHERTEXT_LEN` | 768 | Static KEM-512 ct |
| `NOISE_RESPONSE_CIPHERTEXT_LEN` | 1088 | Ephemeral KEM-768 ct |
| `NOISE_PK_HASH_LEN` | 32 | SM3 输出 |
| `NOISE_TIMESTAMP_LEN` | 12 | u64 + u32 |
| `noise_encrypted_len(plain)` | plain + 16 | SM4-GCM tag |

条件编译：`STATIC_KEM_MLKEM512` + `EPHEMERAL_KEM_MLKEM768`（`messages.h` L57–80）。

### 6.2 struct 字段映射

**`message_handshake_initiation`**

| 字段 | 主套件字节 |
|------|-----------|
| header | 4 |
| sender_index (sid_i) | 4 |
| unencrypted_ephemeral (E_i^pub) | 1184 |
| unencrypted_ciphertext (ct₁) | 768 |
| encrypted_static | 48 |
| encrypted_timestamp | 28 |
| macs | 32 |
| **合计** | **2068** |

**`message_handshake_response`**

| 字段 | 主套件字节 |
|------|-----------|
| header + sid_r + sid_i | 12 |
| unencrypted_ephemeral_ct (ct₂) | 1088 |
| unencrypted_static_ct (ct₃) | 768 |
| encrypted_nothing (empty) | 16 |
| macs | 32 |
| **合计** | **1916** |

**`message_handshake_key_confirm`**

| 字段 | 主套件字节 |
|------|-----------|
| header + sid_i + sid_r | 12 |
| encrypted_conf | 16 |
| macs | 32 |
| **合计** | **60** |

### 6.3 报文类型枚举

```c
MESSAGE_HANDSHAKE_INITIATION   = 1
MESSAGE_HANDSHAKE_RESPONSE   = 2
MESSAGE_HANDSHAKE_KEY_CONFIRM = 3
MESSAGE_HANDSHAKE_COOKIE     = 4
```

（`messages.h` L125–131）

## 7. 与 MTU / 分片的关系

| 报文 | 尺寸 | >1400 B | 主套件行为 |
|------|------|---------|-----------|
| Initiation | 2068 | 是 | 应用层分片（magic 0x5051） |
| Response | 1916 | 是 | 同上 |
| Key Confirmation | 60 | 否 | 单 UDP 包 |

阈值 1400 B 由 `PQST_APP_LAYER_FRAGMENTATION` 控制；主套件 `suite_meta.env` 记录 `APP_LAYER_FRAGMENTATION_EFFECTIVE=1`。

详见 [06](06-分片与MTU.md)。
